## 概要

ツバサ（SEO/AIO診断ボット）の機能拡張。現行のSEO診断機能に加え、**AI推薦力診断（ARI強化版）** と **AI格差診断（APL簡易版）** の2機能を実装し、診断→コンサル→導入支援の一貫収益フローを確立する。

---

## Feature A: AI推薦力診断（ARI Enhanced）

### 概要

4大AIモデル（ChatGPT / Gemini / Perplexity / Claude）に並列問い合わせし、対象企業・サービスがAIにどれだけ推薦・認知されているかを100点満点でスコア化する。

### 診断ロジック

```
入力: 企業名 / サービス名 / キーワード
    │
    ▼
並列API呼び出し（4モデル同時）
    ├─ ChatGPT (GPT-4o)      : "〇〇といえばどんな企業/サービスですか？"
    ├─ Gemini (1.5 Pro)      : 同上
    ├─ Perplexity (sonar)    : 同上
    └─ Claude (claude-sonnet): 同上
    │
    ▼
応答解析（レスポンスに企業名が含まれるか・推薦文脈か）
    │
    ▼
ARIスコア算出（100点満点）
    ├─ 認知スコア: 4モデル中何モデルが言及したか (×25点)
    ├─ 推薦スコア: ポジティブ文脈での言及率 (×1.5倍重み)
    └─ 詳細度スコア: 具体的情報（住所・価格・特徴）の言及数
    │
    ▼
レーダーチャート生成 + 業界平均比較
    │
    ▼
LINEでレポート送信（Flex Message）
```

### ARIスコア基準

| スコア | 評価 | 意味 |
|--------|------|------|
| 80〜100 | S: AI推薦上位 | 4モデル全てで積極推薦 |
| 60〜79 | A: 認知確立 | 3モデル以上で言及 |
| 40〜59 | B: 部分認知 | 1〜2モデルで言及 |
| 20〜39 | C: 認知不足 | 言及なし・ネガティブ |
| 0〜19 | D: 要緊急対策 | 全モデルで未認知 |

### コスト見積もり

- 1診断あたり: ~$0.08〜0.12（4モデル並列）
- 月100診断想定: ~$10/月
- 対策: レート制限（1社/日）・キャッシュ24時間・同一企業の再診断は48時間後以降

## Feature B: AI格差診断（APL Simple）

### 概要

LINEチャット経由の8問アンケートで、企業のAI活用レベル（APL: AI Proficiency Level）を0〜6の7段階で診断。業界平均との比較と個別改善アドバイスを提供し、HSビルのコンサルティングサービスへのCTAに繋げる。

### 診断フロー

```
LINEから「AI診断」と送信
    │
    ▼
ウェルカムメッセージ + 同意確認
    │
    ▼
質問フェーズ1（3問 → 簡易プレビュー）
    ├─ Q1: AI活用ツールを業務で使っていますか？ (0-3点)
    ├─ Q2: 社内でAIルールや方針を定めていますか？ (0-3点)
    └─ Q3: AI活用で具体的な成果が出ていますか？ (0-3点)
    │
    ▼
中間フィードバック（「あなたの暫定APLは〇です。続けて詳細診断しますか？」）
    │
    ▼
質問フェーズ2（5問 → 本格診断）
    ├─ Q4: AIによる業務自動化の範囲は？ (0-3点)
    ├─ Q5: AIデータ分析を意思決定に活用していますか？ (0-3点)
    ├─ Q6: 自社AIモデルや独自プロンプトがありますか？ (0-3点)
    ├─ Q7: AI活用のKPIを設定・計測していますか？ (0-3点)
    └─ Q8: 競合他社との比較でAI活用は？ (0-3点)
    │
    ▼
APLレベル算出（合計0〜24点 → レベル0〜6）
    │
    ▼
結果レポート配信（Flex Message）
    ├─ APLレベルと説明
    ├─ 業界平均との比較グラフ
    ├─ トップ3改善アドバイス
    └─ CTA: 「無料相談を予約する」→ マルモくん予約フロー
```

### APLレベル定義

| APL | 点数 | レベル名 | 典型的な企業像 |
|-----|------|---------|--------------|
| 0 | 0〜3 | 未導入 | AI未活用・関心なし |
| 1 | 4〜6 | 探索期 | ChatGPT個人利用あり |
| 2 | 7〜9 | 初期導入 | 特定業務でAI試用中 |
| 3 | 10〜13 | 部分活用 | 複数部門でAI活用 |
| 4 | 14〜17 | 組織統合 | AI方針・KPI設定済み |
| 5 | 18〜21 | 高度活用 | 自動化・分析に本格活用 |
| 6 | 22〜24 | AI先進企業 | 独自AI・継続改善サイクル確立 |

## 技術仕様

### スタック

| レイヤー | 技術 | 備考 |
|---------|------|------|
| API基盤 | FastAPI (Python) | 既存ツバサのExtension |
| 並列処理 | asyncio + httpx | 4モデル同時呼び出し |
| データ永続化 | SQLAlchemy + SQLite/PostgreSQL | 診断履歴・スコア管理 |
| LINE連携 | LINE Messaging API (Webhook) | Flex Message v2 |
| キャッシュ | Redis or インメモリキャッシュ | 24時間ARIキャッシュ |
| ビジュアライゼーション | matplotlib → PNG → LINE送信 | レーダーチャート |

### API統合

```python
# Feature A: 4モデル並列呼び出しイメージ
async def query_all_models(company_name: str) -> dict:
    tasks = [
        query_openai(company_name),      # GPT-4o
        query_gemini(company_name),      # Gemini 1.5 Pro
        query_perplexity(company_name),  # Perplexity sonar
        query_claude(company_name),      # Claude Sonnet
    ]
    results = await asyncio.gather(*tasks, return_exceptions=True)
    return calculate_ari_score(results)
```
