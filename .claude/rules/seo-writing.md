# SEOライティングルール

## 適用範囲
**/.md (content/blog/ 配下) **/-seo.md


## ブログ記事の必須納品セット
公式サイトブログ記事は、以下を1セットとして必ず納品すること：

### 1. メタ情報
- **メタタイトル**: 30〜35文字、主要キーワード含有、クリック率を意識
- **メタディスクリプション**: 120〜160文字、検索意図に沿った要約、CTA含有
- **記事タイトル（H1）**: メインキーワード含有、読者の関心を引く表現、1記事1つ

### 2. 本文
- イントロ（読者の課題提示、100〜150文字）
- 本文（H2見出しで構成、必要に応じてH3）
- まとめ（要点再確認＋CTA）
- 本文内FAQ（記事内容に関連する質問と回答を本文中に自然に組み込む）

### 3. AIスタッフ活用
- 記事テーマに合ったAIスタッフ（エリカ / マルモ / ツバサ）を選定
- AIスタッフの口調・キャラクターで1〜2箇所コメントやワンポイントを挿入
- 参照: `ai-staff/{name}/persona.md`

### 4. イメージ画像生成プロンプト
- 記事テーマに沿った画像生成プロンプト（日本語＋英語）を記事ごとに作成
- アイキャッチ用1枚 + 本文挿入用1〜2枚
- HSビルの雰囲気（静寂・集中・プロフェッショナル）を反映
- AIスタッフが登場する場合はペルソナに準拠した外見描写

### 5. 構造化マークアップ（JSON-LD）
- Article構造化データ（著者、公開日、画像、組織情報）
- FAQPage構造化データ（本文FAQ対応、3〜5問）
- LocalBusiness構造化データ（該当する場合）
- BreadcrumbList構造化データ

### 6. 構造化FAQ
- 本文内FAQとは別に、JSON-LD形式のFAQPageマークアップを出力
- Google検索結果のリッチリザルト表示を狙う
- 質問は検索クエリを意識した自然な表現にする

## 基本ルール

### E-E-A-T原則の徹底
- **Experience（経験）**: 実際の運営データ、導入事例、利用者の声を必ず含める
- **Expertise（専門性）**: 専門用語は正確に使用し、必要に応じて解説を加える
- **Authoritativeness（権威性）**: メディア掲載実績、AI横断監査82点、Google口コミ★4.4等を引用
- **Trustworthiness（信頼性）**: 誇張表現を避け、具体的な数値と事実に基づく情報提供

### AIO（AI Overviews）対策
- 質問形式の見出しを活用（H2レベル）
- 簡潔で明確な回答を冒頭に配置（AIが引用しやすい構造）
- 構造化されたリスト、表を積極的に使用
- FAQ形式のセクションを含める
- 一次情報（実データ、料金、住所）を本文中に明示

### A2A（Agent-to-Agent）対策
- Agent Card、llm.txt への言及を適宜含める
- 機械可読なエンドポイント情報を記事中に記載可
- 外部AIエージェントからの発見性を高める構造
