# SEOライティングルール

## 適用範囲
```
**/*.md (content/blog/ 配下)
**/*-seo.md
```

## 基本ルール

### E-E-A-T原則の徹底
- **Experience（経験）**: 実際の利用体験、具体的な事例を必ず含める
- **Expertise（専門性）**: 専門用語は正確に使用し、必要に応じて解説を加える
- **Authoritativeness（権威性）**: 公式情報、信頼できるソースへのリンクを含める
- **Trustworthiness（信頼性）**: 誇張表現を避け、事実に基づいた情報提供

### AIO（AI Overviews）対策
- 質問形式の見出しを活用（H2レベル）
- 簡潔で明確な回答を冒頭に配置
- 構造化されたリスト、表を積極的に使用
- FAQ形式のセクションを含める

### コンテンツ構造
```markdown
# メインキーワードを含むH1タイトル（1記事1つ）

## イントロダクション
- 読者の課題・疑問を提示
- 記事で得られる情報を明示
- 100〜150文字程度

## 本文（H2見出しで構成）
### 詳細説明（必要に応じてH3使用）

## まとめ
- 要点の再確認
- CTA（行動喚起）

## FAQ（任意）
```

### キーワード戦略
- 自然な文脈でキーワードを配置（詰め込み禁止）
- ロングテールキーワードを意識
- 地域キーワード（神戸、三宮、兵庫）を適切に含める

### 内部リンク
- 関連する knowledge/ ファイルの内容を参照
- 関連記事への自然なリンク挿入
- アンカーテキストは具体的に

### 禁止事項
- 誇大広告・誤解を招く表現
- 根拠のない効果の主張
- 競合他社の誹謗中傷
- コピーコンテンツ

## 品質チェックリスト
- [ ] タイトルに主要キーワードが含まれているか
- [ ] イントロで読者の課題を明示しているか
- [ ] E-E-A-Tを意識した内容か
- [ ] 見出し構造は適切か（H1→H2→H3の階層）
- [ ] 内部リンクを2〜3個以上含んでいるか
- [ ] 画像にalt属性が設定されているか
- [ ] まとめとCTAがあるか
- [ ] 誤字脱字がないか
