# コンサルティング提案書作成ルール

## 適用範囲
```
consulting/proposal-templates/**/*.md
consulting/active-projects/**/*.md
```

## 基本原則

### プロフェッショナリズム
- ビジネス文書として適切な丁寧語・敬語を使用
- 論理的で説得力のある構成
- データ・数値に基づいた提案
- 専門用語は必要に応じて解説を付ける

### カスタマイズ
- クライアント企業の業種・規模に応じた提案
- ヒアリング内容を反映した個別最適化
- 汎用的なテンプレートそのままの使用禁止

### 参照リソース
```
@consulting/proposal-templates/ai-helpdesk.md
@consulting/case-studies/
@consulting/roi-models/
@knowledge/business-pillars.md
```

## 提案書の構成

### 1. エグゼクティブサマリー
- A4 1枚以内に要約
- 課題、提案、期待効果を簡潔に
- 経営層が最初に読む部分を意識

### 2. 現状分析
- ヒアリング結果の整理
- 課題の具体的な定量化
- 業界ベンチマークとの比較（可能な場合）

### 3. 提案ソリューション
- 具体的な機能・仕様
- システム構成図・フロー図
- 導入ステップの明示

### 4. 期待効果・ROI
- 定量効果（コスト削減額、時間短縮等）
- 定性効果（顧客満足度向上、ブランド価値等）
- ROI試算（`consulting/roi-models/` を参照）

### 5. スケジュール・費用
- フェーズ別のタイムライン
- 明確な費用内訳
- 支払い条件

### 6. サポート体制
- 導入後のサポート内容
- 連絡体制
- SLA（可能な場合）

## スタイルガイド

### トーン
- 専門的だが親しみやすく
- 断定的すぎず、かつ自信を持った表現
- クライアントとのパートナーシップを意識

### 数値・データ
- 具体的な数値を優先
- 出典を明記（推計の場合はその旨を記載）
- 誇張しない、現実的な見積もり

### 視覚要素
- 図表を効果的に活用
- 箇条書き・リストで読みやすく
- 重要なポイントは太字で強調

## マルモのキャラクター性
提案書はビジネス文書として専門的に記述しつつ、プレゼンテーション時は以下を意識：
- マルモのペルソナを参照：`ai-staff/marumo/persona.md`
- 説明時は親しみやすく、専門用語を噛み砕いて
- クライアントの理解度に応じて説明レベルを調整

## 守秘義務・コンプライアンス
- クライアント情報の取り扱いに注意
- 他社事例を紹介する場合は許可を得たもののみ
- 過度な約束・保証を避ける

## 品質チェックリスト
- [ ] エグゼクティブサマリーは1枚以内か
- [ ] 課題が定量的に示されているか
- [ ] 提案内容が具体的か
- [ ] ROI試算が現実的か
- [ ] スケジュールが実現可能か
- [ ] 誤字脱字がないか
- [ ] 専門用語に解説がついているか
- [ ] 図表は見やすいか
- [ ] クライアント固有の情報が反映されているか
