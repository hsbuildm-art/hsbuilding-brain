# HSビル マルチエージェント・コンテンツシステム v1.0
> 設計日: 2026-02-18
> 目的: SEO/AIO記事の自動企画→画像生成→投稿を仕組み化し、事業売上に直結させる

## 全体アーキテクチャ

yuki（オーナー）= 週次レビュー → 承認/修正指示のみ（週18分） │ ▼ Agent 0: 司令塔（Claude） ├─ Agent 1: 市場調査（毎週月曜） ├─ Agent 2: 記事執筆（承認後） ├─ Agent 3: 画像生成（記事確定後） ├─ Agent 4: 投稿配信（最終承認後） └─ Agent 5: 分析報告（毎週日曜）


## Agent 1: 市場調査エージェント
- Google Search Console API → 検索クエリ・CTR・掲載順位
- Wix Analytics API → ページ別PV・滞在時間・離脱率
- Google Trends → トレンドKW
- 事業柱別にKWクラスタ分類 → スコアリング → 週次テーマ提案書生成

## Agent 2: 記事執筆エージェント
- テーマに最適なキャラ自動選定（初心者→マルモ、AI→エリカ、経営→ツバサ）
- SEO最適化構成（H2/H3）+ AIO対応（FAQ構造化データ）
- 内部リンク・CTA・クーポンWELCOME10を自動挿入
- メタディスクリプション・OGP・slugを同時生成

## Agent 3: 画像生成エージェント
- キャラリファレンスシートのIdentity Lockプロンプトを基盤
- 記事セクションごとに必要画像を特定（アイキャッチ1枚+本文内2-4枚）
- 生成API: Gemini/nano BananaPro（エリカ・ツバサ）、DALL-E/Midjourney（マルモ）
- SNS用リサイズ版も同時生成

## Agent 4: 投稿配信エージェント
- Wix Blog API でドラフト投稿
- SNS自動配信: Instagram / note / LinkedIn / X
- LINE公式通知 + Google Indexing API

## Agent 5: 分析報告エージェント
- 記事ごとのPV・順位・予約コンバージョン追跡
- 高パフォーマンス記事の横展開提案
- 低パフォーマンス記事のリライト候補選出

- Google Drive「HSビル_AIアセット/」に画像実体を保管
- NotebookLM「HSビル AIキャラ管理」で自動分類・プロンプト管理
- GitHub ai-staff/*/image-index.md にインデックス記録
- Claudeはimage-index.mdを参照して画像選定・プロンプト生成
